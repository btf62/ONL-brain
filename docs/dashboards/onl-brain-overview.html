Source systems with guardrails

A source system can provide evidence without becoming a playground. Gmail and Calendar are especially constrained: narrow reads only, no sending, no scheduling, no broad browsing.

Protected access

Gmail narrow reads, never send

Safety stance

Access does not imply permission to browse.

The governance layer now says agents prepare, summarize, flag, and recommend. Humans decide and act. This lets the repo gain useful context without turning connected systems into unattended automation.
