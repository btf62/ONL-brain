# Technical Systems

This document should become the source of truth for the tools and systems used to run the online campus.

## Current system categories

- Streaming platform
- Online chat platform
- Video capture and production tools
- Scheduling and communication tools
- Rock RMS for next-step and care pipelines
- Analytics and reporting tools

## For each system, document

- Purpose
- Owner
- Access location
- Login or credential process
- Weekly use case
- Common failure points
- Backup plan

## Technical incident log prompts

When something breaks, capture:

- What happened
- When it happened
- Who noticed it
- What the impact was
- How it was resolved
- What change would prevent recurrence

## Next step

As this repo grows, add one file per major system with setup instructions and failure-response notes.
