# Governance Overview

Governance documents define how this repository is maintained as a reliable source of truth.

They answer questions like:

- What belongs in this repo?
- What belongs in Confluence?
- Which system is canonical for a given kind of information?
- How do we decide when repo content should be promoted into Confluence?
