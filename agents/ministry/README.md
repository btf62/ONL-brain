# Ministry Agents

This folder contains agent definitions for software workers that support online campus ministry operations.

These agents may inspect source systems, summarize context, flag risks, prepare drafts, and suggest next actions.

They do not replace pastoral judgment, staff ownership, or serve team responsibility.

- [Source Digest Agent](source-digest-agent.md)
- [Online Response Queue Agent](online-response-queue-agent.md)
- [Baptism Pipeline Agent](baptism-pipeline-agent.md)
- [Confluence Promotion Agent](confluence-promotion-agent.md)
- [Serve Team Roster Agent](serve-team-roster-agent.md)
- [Budget Watch Agent](budget-watch-agent.md)

## Shared operating rules

- Read the relevant repo docs before making recommendations.
- Prefer source-system evidence over memory.
- Follow [Access and Safety](../../docs/governance/access-and-safety.md).
- Follow [Source Systems](../../docs/governance/source-systems.md) boundaries, especially Gmail and Google Calendar access limits.
- Distinguish current source data from current ministry reality when they differ.
- Never expose sensitive pastoral, financial, or personal details unnecessarily.
- Recommend and prepare; humans decide and act.
- Never send email to anyone other than Brad's own work account when he explicitly requests a self-email.
- Never schedule calendar events or modify calendar items.
